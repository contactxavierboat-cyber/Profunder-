PROFUNDR — Bureau Prediction Engine

1️⃣ The Goal

When a user selects a lender, Profundr estimates:

• Which bureau will likely be pulled
• How likely that bureau is to be used
• Whether the user’s profile is strong on that bureau

Example output inside Profundr:

Amex Blue Business Cash
	•	Likely Bureau: Experian
	•	Pull Probability: ~80% Experian / 15% Equifax / 5% TransUnion
	•	User Experian Strength: Strong
	•	Recommendation: Safe to apply

⸻

2️⃣ Why This Matters

Each lender often favors a bureau depending on:
	•	State/region
	•	Card type
	•	Time period (banks sometimes change behavior)

If a user has many inquiries on Experian, Profundr might suggest:

Apply for a TransUnion-heavy lender first.

This helps prevent unnecessary denials and protects the user’s credit profile.

⸻

3️⃣ Data the Engine Uses

The prediction combines several signals:

A. Historical patterns

Common lender tendencies (example patterns many users report):
	•	Many American Express cards → often Experian
	•	Many Chase cards → often Experian
	•	Some U.S. Bank products → often TransUnion
	•	Some Barclays products → often TransUnion

(Actual patterns vary by time and region.)

⸻

B. User location

Some lenders pull different bureaus depending on the state.

Example logic:

If state = Texas
Chase → Experian probability increases


⸻

C. Recent datapoints

If the Funding Radar detects recent approvals using a different bureau, the model adjusts.

Example:

Recent posts show:
Barclays pulling Equifax in TX

Probability update:
Equifax 60%
TransUnion 35%
Experian 5%

6️⃣ Integration With the Funding Sequence Engine

Once the bureau predictions are known, Profundr can optimize application order.

Example strategy:

Funding Sequence

3️⃣ Experian lender

This reduces the chance of multiple inquiries on one bureau too quickly.
